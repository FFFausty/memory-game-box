# 模式选择系统使用说明

## 功能概述

已成功创建了模式选择系统，包含以下功能：

- **三种游戏模式**：单机模式、AI对战、本地联机
- **单选机制**：只能选择一个模式，不能重复点击取消选择
- **视觉反馈**：选中模式会放大、颜色变化、添加阴影效果
- **默认选择**：单机模式为默认选择
- **集成检测**：点击开始游戏时会检测当前选择的模式

## 组件文件

1. **ModeSelector.ts** - 模式选择器组件
   - 位置：`assets/scripts/ModeSelector.ts`
   - 功能：管理三种游戏模式的选择逻辑和视觉效果

2. **MainMenu.ts** - 主菜单组件（已更新）
   - 位置：`assets/scripts/MainMenu.ts`
   - 功能：集成模式选择器，在开始游戏时检测模式

## 在场景中配置ModeSelector

### 步骤1：创建模式选择器UI节点

在MainMenu场景中创建模式选择器的UI结构：

```
Canvas
├── ModeSelector (空节点，添加ModeSelector组件)
│   ├── ModeButton1 (按钮节点，包含Label和Sprite)
│   ├── ModeButton2 (按钮节点，包含Label和Sprite)
│   ├── ModeButton3 (按钮节点，包含Label和Sprite)
│   └── ModeDescription (描述文本节点，包含Label)
```

### 步骤2：配置ModeSelector组件

1. 在Canvas下创建一个空节点，命名为"ModeSelector"
2. 为该节点添加"ModeSelector"组件
3. 在组件属性中配置：
   - **Mode Buttons**：将三个模式按钮节点拖拽到数组中
   - **Mode Description**：将描述文本节点拖拽到该属性
   - **Selected Color**：选中时的颜色（默认：蓝色）
   - **Normal Color**：未选中时的颜色（默认：灰色）

### 步骤3：配置MainMenu组件

1. 找到MainMenu节点上的MainMenu组件
2. 将ModeSelector节点拖拽到"Mode Selector"属性中

## 模式选择器功能特性

### 单选机制
- 点击任意模式按钮会选中该模式
- 已选中的模式不能通过重复点击取消
- 点击其他模式会自动取消之前的选择

### 视觉反馈
- **选中状态**：放大1.1倍、颜色变为蓝色、添加阴影效果
- **悬停效果**：鼠标悬停时轻微放大（仅未选中状态）
- **平滑过渡**：所有变化都有0.3秒的动画过渡

### 模式描述
- 选择不同模式时，描述文本会相应更新
- 单机模式："单人游戏，挑战自我"
- AI对战："与人工智能对战"
- 本地联机："与朋友本地对战"

## 游戏启动流程

1. 用户选择游戏模式
2. 用户点击游戏入口的"开始游戏"按钮
3. MainMenu组件检测当前选择的模式
4. 根据模式加载对应的游戏场景
5. 将选择的游戏和模式保存到全局变量中

## 全局变量

游戏启动后，以下信息会保存到全局变量中：

```javascript
window.selectedGame = 'memoryGame'; // 选择的游戏ID
window.selectedGameMode = 'SingleMode'; // 选择的游戏模式
```

## 扩展建议

1. **场景差异化**：可以为不同模式创建不同的游戏场景
2. **模式特定逻辑**：在游戏场景中根据模式加载不同的游戏逻辑
3. **UI优化**：可以添加模式图标、难度选择等扩展功能

## 注意事项

- 确保ModeSelector节点在场景层级中正确配置
- 模式按钮需要包含Label组件用于显示文字
- 模式按钮建议包含Sprite组件用于颜色变化效果
- 描述节点需要包含Label组件用于显示模式描述
